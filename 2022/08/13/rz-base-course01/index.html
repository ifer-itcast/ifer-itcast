01_人资后台

发表于 2022-08-13 更新于 2022-11-25

今日目标

✔ 了解 vue-element-admin。

✔ 了解项目结构和重要文件的作用。

项目介绍

项目名称：人力资源管理系统。是企业的后台管理系统，对企业内部信息进行管理。

项目演示：http://ihrm-java.itheima.net/#/login

项目技术：基于 vue-admin-template 二次开发（不是从 0 开始），使用 ES2015+、vue、vuex、vue-router 、vue-cli 、axios 和 element-ui 等技术。

业务难点：权限控制（不同的人进入系统可以看到的菜单是不同的），表单校验，复杂数据处理，excel 导入导出，国际化。

vue-element-admin

目标

学习和了解通用的 Vue 后台集成方案 vue-element-admin，这是官网。

内容

vue-element-admin 是一个后台前端解决方案，它基于 Vue 和 element-ui 实现，内置了 i18 国际化解决方案，动态路由，权限验证，提炼了典型的业务模型，提供了丰富的功能组件，最终效果。

小结

二开项目：在已有项目的基础之上做开发；vue-element-admin 是一套功能非常强大的库。

vue-admin-template

目标

下载熟悉 vue-admin-template。

内容

集成方案里面包含了所有的业务功能和场景，并不适合直接拿来进行二次开发，因为要删减的东西太多了，可以用来学习和使用。

集成方案不适合我们直接拿来进行二次开发，基础模板 则是一个更好的选择，它包含了基本的登录 / 鉴权 / 主页布局等一些基础功能模板，我们可以直接在该模板上进行功能的扩展和项目的二次开发。

这个是基础模板的在线链接。

小结

  • vue-admin-template 是在 vue-element-admin 的基础上做了大量的删减，只保留了后台管理系统最关键的框架部分，非常适合我们在此基础上做二次开发。

  • 市面上使用 Vue 技术栈开发的 PC 后台项目很多都是基于 vue-admin-template 进行的二次开发，我们的人资项目也基于它来实现。

API 模块单独封装

我们习惯性的将所有的网络请求放置在 api 目录下统一管理，按照模块进行划分，例如 api/user.js，使用了封装的 request 工具，每个接口的请求都单独导出了一个方法，这样做的好处就是，任何位置需要请求的话，可以直接引用。

为了后续更好的开发，我们可以先将 api/user.js 代码的方法设置为空，后续在进行更正。

配置网站端口号和标题

目标

设置统一的本地访问端口和网站 title。

端口

在正式开发业务之前，先将项目的本地端口和网站名称进行一下调整，vue.config.js。

  • process.env 是 Node 的语法，可以获取到环境变量。

  • 执行 npm run dev，此时会加载执行 .env.development 文件内容。

  • 执行 npm run build:prod，会加载执行 .env.production 文件内容。

  • 所以，如果想要设置开发环境的接口，直接在 .env.development 中写入对于 port 变量的赋值即可。

🤔 注意：修改服务的配置文件，想要生效的话，必须要重新启动服务，值 8888 后面不能留有空格。
