组件的概念

目标

了解组件。

讲解

  • 组件概念：组件是可复用的 Vue 文件，是页面中的一部分，内部封装了模板、样式、JS 代码。

  • 组件化：就是采用分而治之的思想，把复杂页面拆分成一个个的组件单元，各自独立又可组合。

小结

组件是什么?

答案
  • 可复用的 Vue 文件，是页面中的一部分。

什么时候封装组件?

答案
  • 遇到重复模板，需要复用的时候。

组件好处?

答案
  • 采用分而治之的思想，把复杂页面拆分成一个个的组件单元，各自独立又可组合。

组件 scoped

目标

解决多个组件之间样式名相同带来的冲突问题。

讲解

需求：在 Panel.vue 中 <div class="panel"></div>，通过 .panel 选择器，设置背景色。

问题：发现也影响到了 App.vue 中的 .panel 样式。

解决：给 Panel.vue 组件里 style 标签上加 scoped 属性即可。

在 style 上加入 scoped 属性，就会在此组件的所有标签上加上一个随机生成的 data-v 开头的属性，编译之后的样式也类似 div[data-v-xxx] 的形式。

小结

Vue 组件内样式，只针对当前组件内标签生效如何做?

答案
  • 给 style 上添加 scoped。

原理和过程是什么?

答案
  • 会自动给标签添加 data-v-hash 值属性，所有选择都带属性选择。

关于单向数据流

目标

掌握单项数据流的概念和特点。

内容

  • 概念，单向数据流，也叫做自上而下的数据流，它有两个特点，如下。

    a，当父组件中的数据更新时，子组件接收到的数据也会自动更新。

    b，子组件不能直接去修改父组件的数据。

    c，类比：就像瀑布的水一样只能从上往下流动，并且，当上游的水变浑浊，下游的水也会受到影响。

  • 为什么需要这样？对于大型项目来说为了方便管理数据，追溯错误的来源。

拆分

简单数据类型，一定不能改，否则会有警告，也会导致子组件的数据和父组件不一致的问题，例如。

小结

为何不建议，子组件修改父组件传过来的数据？

答案
  • 可能会导致父子数据不一致的问题；不方便数据管理和追溯错误。

单向数据流的两个特点是什么？

答案
  • 当父组件中的数据更新时，子组件接收到的数据也会自动更新。
  • 子组件不能直接去修改父组件的数据。

props 里定义的变量能修改吗?

答案
  • 简单数据类型，一定不能改，否则会有警告，也会导致子组件的数据和父组件不一致的问题。
  • 复杂数据类型，引用不能改，内容可以改，即便内容可以改，也不建议改，因为要遵循单项数据流的思想。

数据切换

目标

点击底部按钮切换数据。

讲解

  • 需求：点击底部按钮高亮并切换数据。

  • 分析。

    1. TodoFooter.vue 组件定义 isSel，默认值为 all，yes，no 其中一种。

    2. 动态给每一个 a 标签绑定 class，如 :class="{ selected: isSel === 'all' }"。

    3. 点击 a 标签时修改 isSel 的值为当前点击的项。

    4. 给 ul 绑定点击事件（点击 a 的时候会冒泡到这儿），通过子传父，把 isSel 传到父组件 App.vue。

    5. 父组件根据接收到的 isSel 来修改自己内部的变量 getSel。

    6. 父组件定义计算属性 showArr，根据 getSel 决定从 list 里筛选出哪些数据。

    7. 把筛选后的结果重新传递给 TodoMain 和 TodoFooter 组件。
