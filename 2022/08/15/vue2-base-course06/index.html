06_Vue 基础

发表于 2022-08-15 更新于 2022-09-02

今日学习目标

✔ 能够了解单页面应用概念和优缺点。

✔ 能够掌握 vue-router 路由系统使用。

✔ 能够掌握声明式导航和编程式导航用法。

✔ 能够掌握路由嵌套和路由守卫。

路由是什么

目标

了解什么是路由。

讲解

  1. 生活中的路由器：设备和 IP 的映射关系。

  1. 后端路由：接口和服务的映射关系。

  1. 前端路由：路径和组件的映射关系。

小结

路由是什么呢？

答案
  • 路由是一种映射关系。

Vue 中的路由是什么？

答案
  • 路径和组件的映射关系。

路由为何用

目标

了解为何要学习和使用路由。

讲解

  • 单页面应用（SPA）：所有功能在一个 HTML 页面上，例如网易云音乐。

  • 如何做到不同功能间切换呢？答案是：前端路由。

  • 优点。

    • 数据传递容易，开发效率高。

    • 整体不刷新页面，用户体验更好。

  • 缺点。

    • 开发成本高（需要学习专门知识）。

    • 首次加载会相对慢一些；不利于 SEO。

小结

什么是单页面应用？

答案
  • 所有的业务都在一个页面编写，只有一个 HTML。

单页面应用好处？

答案
  • 开发效率高，用户体验好。

单页面如何切换场景？

答案
  • 依赖路由切换显示。

Vue Router 介绍

目标

如何在 Vue 项目中集成路由。

讲解

Vue Router 官网。

  • 和 Vue 深度集成；

  • 可以定义映射规则；

  • 模块化；

  • 提供 2 个内置全局组件（入口和出口）；

  • 声明式导航自动激活的 CSS class 的链接；

  • …

小结

Vue 中如何实现路由？

答案
  • 集成 vue-router 模块包。

Vue 组件分类

目标

.vue 文件分 2 类，一个是路由组件，一个是普通组件。

讲解

  • 路由组件，通过路由跳转访问到的组件，一般放在 src/views（pages） 文件夹。

  • 普通组件，在路由组件中使用的组件，一般可以用来复用，一般放在 src/components 文件夹。

小结

什么是路由组件，放在哪？

答案
  • 通过路由跳转访问到的组件，一般放在 src/views（pages） 文件夹。

什么是普通组件，放在哪？

答案
  • 在路由组件中使用的组件，一般可以用来复用，一般放在 src/components 文件夹。

Vue Router 使用

目标

学会 Vue 官方提供的 vue-router 路由系统功能模块使用。

讲解

  1. 准备页面结构和样式，App.vue。
  1. 安装 Vue Router。
  1. 新建 router/index.js，导入路由并注册。
  1. 创建路由组件和对应的路由规则数组。
  1. 创建路由对象，传入规则，并默认导出。
  1. 关联到 Vue 实例，main.js。
  1. 指定路由的入口和出口，App.vue。

小结

Vue Router 本质是什么？

答案
  • 第三方包，下载后集成到 Vue 项目中。

Vue Router 如何使用？

答案
  • 下包/引入/注册/规则/路由对象/注入/挂载点。

规则如何生效？

答案
  • 切换 url 上 hash 值，开始匹配规则，对应组件展示到 router-view 位置。

Vue 路由-声明式导航-跳转

目标

可用全局组件 router-link 来替代 a 标签

讲解

  • Vue Router 提供了一个全局组件 router-link。

  • router-link 最终会渲染成 a 链接，to 属性等价于 a 标签的 href 属性（to 无需 #）。

  • router-link 提供了声明式导航激活时的类名（可以做高亮）。

App.vue

小结

router-link 是什么？

答案
  • VueRouter 提供的，内置的全局组件，本质就是 a 标签。

router-link 配合哪个属性跳转？

答案
  • to 属性。

router-link 好处？

答案
  • 自带激活时的类名，可以做高亮。

Vue 路由-声明式导航-传参

目标

在路由跳转时，可以给路由对应的组件内传递数据。

讲解

  • 在 router-link 的 to 属性传值，语法格式如下。

    • /path?参数名=值。

    • /path/值，需要路由配置时指定 path: "/path/参数名"。

  • 对应理由组件内接收传递过来的值。

    • $route.query.参数名。

    • $route.params.参数名。

  1. 创建 views/Part.vue，准备接收路由上传递的参数和值。
  1. 路由定义，router/index.js。
  1. App.vue。

小结

声明式导航跳转时，如何传值给路由页面？

答案
  • to="/path?key=value"。
  • to=“/path/value”（需在路由规则里配置 /path/:key）。

如何接收路由传值？

答案
  • $route.query.参数名。
  • $route.params.参数名。

Vue 路由-重定向

目标

匹配 path 后，强制切换到其他 path 上。

讲解

  • 网页打开 url 默认 hash 值是 / 路径。

  • redirect 表示要重定向到哪个路由路径。

需求: 网页默认打开，匹配路由 “/“，强制切换到 “/find” 上。

🤔 强制重定向后，还会重新来数组里匹配一次规则。

小结

通过哪个属性进行路由重定向？

答案
  • redirect 配置项，值为要切换的路由地址。

Vue 路由-404 页面

目标

如果路由 hash 值，没有和数组里的路由规则匹配，期望展示 404 页面。

讲解

语法：路由规则的最后，path 对应的值是 *，表示前面不匹配就命中最后这个规则，展示对应的组件。

  1. 创建 views/NotFound.vue 页面。
  1. 在 main.js 修改路由配置。

小结

如何给路由体系里设置 404 页面？

答案
  • 在数组最后一个位置，插入匹配 * 的规则，展示 404 页面。

通过哪个配置项可以修改路由模式呢？

  • mode

Vue 路由-编程式导航-跳转

目标

用 JS 代码来进行跳转。

讲解

  1. 给路由起名字，router/index.js。
  1. App.vue，换成 span 标签，配合 JS 的编程式导航跳转。

小结

编程式导航根据哪两个可以进行跳转？

答案
  • name 和 path。

Vue 路由-编程式导航-传参

目标

目标：JS 跳转，路由传参。

讲解

  • 推荐：name + query 或 name + params。

  • 其他：path + query。

  • 注意：使用 path 会自动忽略 params。

App.vue

小结

编程式导航传参推荐的方式有哪些?

答案
  • name + query 或 path + query 传，$route.query 接。
  • name + params 传，$route.params 接。

在现有的一级路由下，再嵌套二级路由。

网易云音乐，发现音乐下的二级路由演示。

  • 创建需要用的所有组件。

    views/Second/Recommend.vue，发现音乐页 => 推荐页面。

    views/Second/Ranking.vue，发现音乐页 => 排行榜页面。

    views/Second/SongList.vue，发现音乐页 => 歌单页面。

  • 配置二级路由的入口和出口，views/Find.vue。

小结

通过哪个属性配置二级路由？

答案
  • children。

Vue 路由-声明式导航-类名区别

目标

目标：router-link 自带的 2 个类名的区别是什么。

讲解

观察路由嵌套导航的样式。

  • router-link-exact-active (精确匹配) ：url 中的 hash 值路径，与 href 属性值完全相同，会有此类名。

  • router-link-active (模糊匹配)：url 中 hash 值，是以 href 属性值开头的就可以。

小结

自动添加的 2 个类名的区别?

答案
  • router-link-exact-active，url 中的 hash 值路径，与 href 属性值完全相同，会有此类名。
  • router-link-active，url 中 hash 值，是以 href 属性值开头的就可以。

Vue 路由-全局前置守卫

目标

路由跳转之前，先执行一次前置守卫函数，判断是否可以正常跳转。

讲解

使用例子：在跳转路由前，判断用户登陆了才能去【我的音乐】页面，未登录弹窗提示回到【发现音乐】页面。

在路由对象上使用固定方法 beforeEach。

小结

什么是路由守卫？

答案
  • 路由在真正跳转前，会执行 beforeEach 函数，可以在内部根据登录状态和路由地址来决定是否放行。

Vue 路由-组件缓存

目标

学习 keep-alive，实现组件缓存效果（当组件看不见时，把组件相关信息存储在内存中）。

讲解

  1. 给 3 个子组件添加 created 和 destroyed 方法。

  2. 测试切换效果。

  1. 给 router-view 外面包裹 Vue 自带的 keep-alive 标签即可。

小结

路由页面切换时，都会执行哪些生命周期方法？

答案
  • 消失：destroyed。
  • 出现：created。

如何缓存组件到内存里，提升切换效率？

答案
  • 用 keep-alive，包裹切换的挂载点即可。

Vue 路由-组件缓存-匹配缓存

目标

学习 include/exclude，区别缓存组件。

讲解

需求: 只缓存【发现音乐】页面。

  1. 需要给要区分的组件，设置 name 字段和值。
  1. 在 keep-alive 的时候，使用 include/exclude 区分即可。

小结

keep-alive 如何区别缓存哪些组件？

答案
  • 在 keep-alive 使用 include 选项和 exclude 选项。

Vue 路由-组件缓存-新钩子函数

目标

使用 keep-alive 后，新增了 2 个钩子函数。

讲解

一旦进行了缓存，组件不执行初始化创建或销毁的方法了，如何知道组件激活/失活呢？

  1. activated，组件被激活触发的钩子。

  2. deactivated，组件失活触发的钩子。

小结

加上 keep-alive 以后，如何知道组件激活/非激活呢?

答案
  • 设置 2 个新的钩子函数，activated 和 deactivated。
