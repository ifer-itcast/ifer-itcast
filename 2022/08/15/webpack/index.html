关于 Yarn

了解 yarn

  • 快速、可靠、安全的依赖管理工具，作用类似于 npm，中文官网地址。

  • 注意：系统原因，如果 yarn 实在安装不上可以先用 npm。

下载 yarn

下载地址: https://yarn.bootcss.com/docs/install/#windows-stable

  • windows，软件包（在笔记文件夹里），不建议安装到中文路径下。

  • mac，通过命令安装（也可还用 npm）。

如果上面不行，试试这个。

使用 yarn

与 npm 类似，新建一个空白文件夹，可以执行以下命令尝试一下。

  1. 初始化，得到 package.json 文件（终端路径所在文件夹下）。
  1. 添加依赖（下包）。
  1. 移除包。
  1. 安装项目全部依赖（一般拿到别人的项目时，缺少 node_modules）。
  1. 全局。

Webpack 概念了解

目标

知道什么是 Webpack？

场景

开发完项目，可以用 Node + Webpack 来分析、翻译、压缩、打包，加快浏览器打开速度。

概念

Webpack 是一个打包构建工具，能对代码进行压缩、合并、转换等操作，减小代码体积，让浏览器更快速打开网页。

小结

什么是 Webpack？

答案
  • Webpack 是一个打包构建工具。
  • 能对代码进行压缩、合并、转换等操作。
  • 减小代码文件体积，让浏览器更快速打开网页。

Webpack 处理 CSS

目标

学习 loader 加载器，打包更多类型文件。

步骤

style-loader，css-loader。

  1. 安装依赖。
  1. webpack.config.js 配置。
  1. 执行打包命令，观察打包后 dist 下。

效果

  1. CSS 代码被打包进了 dist/bundle.js 中。

  2. 运行时，CSS 代码插入到了 HTML 的 style 标签中。

小结

Webpack 如何支持 CSS 打包？

答案
  • 使用 style-loader 和 css-loader。

style-loader 和 css-loader 作用?

答案
  • css-loader 让 Webpack 识别 .css 文件，打包代码到 JS 中。
  • style-loader 把 JS 中的 CSS 代码，插入到 style 标签里显示。

打包后样式在哪里？如何生效?

答案
  • 打包后样式在 .js 文件中
  • 运行后，被插入到 style 标签里

Webpack 如何支持图片打包？

对图片有哪两种处理方案？

Webpack 开发服务

目标

  • 每次修改代码，重新打包，才能看到最新的效果，麻烦；实际工作中，打包非常费时，影响开发效率。

  • 下载 webpack-dev-server，启动一个开发服务器，用于快速开发应用程序。

步骤

  1. 下载包。
  1. 配置自定义命令 serve。
  1. 运行命令，启动 webpack 开发服务器。

  1. 效果：以后改 src 下的代码，自动打包更新到浏览器上。

  2. webpack.config.js 中添加服务器配置（配置完毕，记得重启才能生效）。

小结

如何用 Webpack 开发服务器，实时打包我们的代码？

答案
  • 下载 webpack-dev-server 包。
  • 在 package.json 配置 serve 命令，启动。
  • webpack-dev-server 给我们一个地址 + 端口，供浏览器访问查看 index.html 页面和打包后的 JS 和 CSS 等。

如何修改 Webpack 开发服务器的配置呢？

答案
  • 去文档查找配置项的名字。
  • 在 webpack.config.js 的 devServer 选项里添加。

Webpack 打包发布

目标

项目做完了，要上线怎么办。

步骤

  1. 开启 production 模式。

  1. 执行之前的 yarn build 产生 dist 目录。

  2. 把 dist 目录交给后台/运维，部署给客户使用即可（开发环境的代码不用发）。

  3. 准备一个后端 web 服务，把 dist 放进去，暴露成静态资源目录供别人访问。

小结

项目分哪 2 个环境?

答案
  • 线上和线下 2 个环境。
  • 线上也叫"生产环境"/"部署"，英文 "production"。
  • 线下也叫"开发环境"/写代码，英文 "development"。
