事件绑定

概念

  1. 什么是事件：事件是渲染层到逻辑层的通讯方式，通过事件可以将用户在渲染层产生的行为，反馈到逻辑层进行业务的处理。
  1. 小程序中常用的事件。
类型 事件描述 绑定方式
tap 手指触摸后马上离开，类似于 HTML 中的 click 事件 bindtap 或 bind:tap
input 文本框的输入事件 bindinput 或者 bind:input
change 复选框选中状态改变时触发 bindchange 或 bind:change

bindtap

  1. 通过 bindtap 属性，可以为组件绑定 tap 触摸事件。
  1. 在页面的 .js 文件中定义对应的事件处理函数，事件参数通过形参 event 来接收。

touches 和 changedTouches，当事件回调触发的时候，会收到一个事件对象 event，它的详细属性如下。

属性 类型 说明

条件渲染

条件

block

  1. 可以使用 <block/> 标签将多个组件包装起来，统一使用 wx:if 控制属性。

  2. 注意： <block/> 并不是一个组件，它仅仅是一个包装元素，不会在页面中做任何渲染，只接受控制属性。

hidden

  1. 在小程序框架中，直接使用 hidden="{{ condition }}" 也能控制元素的显示与隐藏。

  2. 案例代码。

if 和 hidden

  1. 运行方式不同。

    • wx:if 以动态创建和移除元素的方式，控制元素的展示与隐藏。

    • hidden 以切换样式的方式（display: none/block），控制元素的显示和隐藏。

  2. 使用建议。

    • 频繁切换时，建议使用 hidden。

    • 控制条件复杂时，建议使用 wx:if，因为可以搭配 wx:elif 和 wx:else 一起使用。

列表渲染

wx:for

  1. 通过 wx:for 可以根据指定的数组，循环渲染重复的组件结构。

wx:key

类似于 Vue 列表渲染中的 :key ，小程序在实现列表渲染时，也建议为渲染出来的列表项指定唯一的 key 值，从而提高渲染的效率。

WXSS

WXSS 和 CSS

  1. 什么是 WXSS（WeiXin Style Sheets）：是一套样式语言，用于描述 WXML 的组件样式。

  2. WXSS 和 CSS 的关系如下。

    • WXSS 具有 CSS 大部分特性，为了更适合开发微信小程序，WXSS 对 CSS 进行了扩充以及修改。

    • 与 CSS 相比，WXSS 扩展的特性主要有 rpx 尺寸单位，也可以使用 @import 样式导入。

掌握 RPX 使用

  • 什么是 RPX（responsive pixel）：是微信小程序独有的屏幕自适应单位，用来解决屏幕适配的单位（目前其他小程序也支持 rpx）。

  • RPX 的实现原理：鉴于不同设备屏幕的大小不同，为了实现屏幕的自动适配，rpx 把所有的屏幕，在宽度上等分为 750 份，即当前屏幕的总宽度为 750 rpx，在较小的设备上，1rpx 所代表的宽度较小，在较大的设备上，1rpx 所代表的宽度较大。小程序在不同的设备上运行的时候，会自动把 rpx 的样式单位换算成对应的像素单位来渲染，从而显示屏幕适配。

RPX 与 PX

  • 以 iPhone6 为例，共有 750 个物理像素，屏幕宽度为 375px，则 750rpx = 750 物理像素 = 375px。

  • 也就是 1rpx = 1 物理像素 = 0.5px。

设备 rpx 换算 px (屏幕宽度/750) px 换算 rpx (750/屏幕宽度)
iPhone5 1rpx = 0.42px 1px = 2.34rpx
iPhone6 1rpx = 0.5px 1px = 2rpx
iPhone6 Plus 1rpx = 0.552px 1px = 1.81rpx

那么也就是说：在 iPhone6 上，如果要绘制宽 100px * 20px 的盒子，换算成rpx单位，宽高分别为 200rpx * 40rpx。官方建议：开发微信小程序时，设计师可以用 iPhone6 作为视觉稿的标准，Phone6 设计稿宽度为 750px，量多少写多少。

@import

使用场景：多个页面存在相同的 CSS 样式时，可以将 wxss 抽取成单独的文件，在每个页面的 wxss 中使用 @import 导入即可。

  1. 定义样式，/common/common.wxss。
  1. 在页面的 wxss 中使用导入样式。

全局样式

定义在 app.wxss 中的样式为全局样式，作用于每一个页面。

局部样式

在 page 的 wxss 文件中定义的样式为局部样式，只作用在对应的页面，并会覆盖 app.wxss 中相同的选择器。

全局配置

全局配置

根目录下的 app.json 文件用来对微信小程序进行全局配置，详细文档，最主要的配置节点如下。

  • pages 数组：配置小程序的页面路径。

  • window 对象：用于设置小程序的状态栏、导航栏、标题、窗口背景色。

  • tabBar 对象：配置小程序的 tab 栏效果。

  • style 对象：是否启用新版样式。

设置导航栏标题

app.json –> window –> navigationBarTitleText。

设置导航栏背景

  1. app.json –> window –> navigationBarBackgroundColor。

  2. 将属性值修改为指定的颜色即可。只可以是 16 进制 RGB 格式，例如 #ff00ff，不支持字符串色值。

导航栏标题颜色

  1. app.json –> window –> navigationBarTextStyle。

  2. 只有 black 和 white 两个选项。

下拉刷新的功能

app.json –> window –> 把 enablePullDownRefresh 的值设置为 true。

设置窗口背景色

当全局开启下拉刷新功能之后，默认的窗口背景为白色，可以通过 app.json -> window -> backgroundColor 进行设置。

下拉 loading 颜色

当全局开启下拉刷新功能之后，默认窗口的 loading 样式为白色。

  1. app.json –> window –> backgroundTextStyle。

  2. 只有两个选项：dark 和 light。

上拉触底的距离

手指在屏幕上的上滑操作，从而加载更多数据的行为，默认是 50px，可以通过 app.json –> window –> onReachBottomDistance 进行配置。

页面配置

页面配置只对当前页面的窗口生效，详细文档。

全局配置和页面配置的关系

  1. 通过 app.json 中的 window 节点，可以全局配置小程序中每个页面的窗口表现。

  2. 如果某些小程序页面，想要拥有特殊的窗口表现，此时在 页面.json 文件中配置即可。

  3. 注意，页面级别配置优先于全局配置生效。

tabBar

tabbar 是移动端应用常见的页面效果，用于实现多页面的快速切换；小程序中通常将其分为底部 tabBar 和顶部 tabBar，tabbar 配置最少 2 个，最多 5 个，当渲染顶部 tabBar 的时候，不显示 icon，只显示文本。

相关配置项

  1. tabBar 组件的属性配置项。
属性 类型 必填 默认值 描述
color HexColor 是 . tab 上的文字默认颜色，仅支持十六进制颜色
selectedColor HexColor 是 . tab 上的文字选中时的颜色，仅支持十六进制颜色
backgroundColor HexColor 是 . tab 的背景色，仅支持十六进制颜色
borderStyle string 否 black tabBar 上边框的颜色， 仅支持 black / white
list Array 是 . tab 的列表，详见 list 属性说明，最少 2 个、最多 5 个 tab
position string 否 bottom tabBar 的位置，仅支持 bottom/ top
custom boolean 否 false 自定义 tabBar
  1. list 节点的配置项。
属性 类型 必填 说明
pagePath string 是 页面路径，必须在 pages 中先定义
text string 是 tab 上按钮文字
iconPath string 否 图片路径，icon 大小限制为 40kb，建议尺寸为 81px * 81px
selectedIconPath string 否 选中时的图片路径，icon 大小限制为 40kb，建议尺寸为 81px * 81px
  1. 不支持网络图片，当 position 为 top 时，不显示 icon。

组成部分实例

如果需要 tabBar 展示到页面中，对应的路径必须有一个是首页。

  1. color：tab 上的文字默认（未选中）颜色。

  2. selectedColor：tab 上的文字选中时的颜色。

  3. backgroundColor：tabBar 背景色。

  4. borderStyle：tabBar上边框的颜色。

  5. iconPath：未选中时的图片路径。

  6. selectedIconPath：选中时的图片路径。

数据请求

小程序网络请求的限制

出于安全性方面的考虑，小程序官方对数据接口的请求做出了如下两个限制。

  • 只能请求 HTTPS 类型的接口。

  • 必须将接口的域名添加到信任列表中。

配置 request 合法域名

  1. 需求描述：假设在自己的微信小程序中，希望请求 https://applet-base-api-t.itheima.net/ 域名下的接口。

  1. 配置步骤：登录微信小程序管理后台 –> 开发 –> 开发设置 –> 服务器域名 –> 修改 request 合法域名。

  2. 修改时需要注意以下事项。

    • 域名只支持 https (request、uploadFile、downloadFile) 和 wss (connectSocket) 协议。

    • 域名不能使用 IP 地址或 localhost。

    • 域名必须经过 ICP 备案。

    • 服务器域名一个月内可申请 5 次修改。

    • 注意：微信公众后台显示一个月可以修改 50 次，这是错误的！一月只能够修改 5 次。

如何发起 GET 网络请求

调用微信小程序提供的 wx.request() 方法，可以发起 GET 数据请求。

如何发起 POST 网络请求

调用微信小程序提供的 wx.request() 方法，可以发起 POST 数据请求。

在页面刚加载时请求数据

在很多情况下，我们需要在页面刚加载的时候，自动请求一些初始化的数据，此时需要在页面的 onLoad 生命周期方法中调用获取数据的函数。

跳过 request 合法域名校验

如果开发前期仅仅提供了 http 协议的接口、暂时没有提供 https 协议的接口，此时为了不耽误开发的进度，我们需要在微信开发者工具中，临时开启【开发环境不校验请求域名、TLS 版本及 HTTPS 证书】选项来跳过服务器域名的校验。此时在微信开发者工具及手机开启调试模式时，则不会进行服务器域名的校验。

🤔 在服务器域名配置成功后，建议开发者关闭此选项进行开发，以确认服务器域名配置正确。

关于跨域和 AJAX 的说明

  1. 跨域问题只存在于基于浏览器的 Web 开发中。由于小程序的宿主环境不是浏览器，而是微信客户端，所以小程序中不存在跨域问题。

  2. AJAX 技术的核心是依赖于浏览器中的 XMLHttpRequest 这个对象，由于小程序宿主环境是微信客户端，所以小程序中不能叫做 发起 AJAX 请求，而是叫做发起网络数据请求。
