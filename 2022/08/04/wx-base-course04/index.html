掌握组件的创建和使用

创建自定义组件

  1. 在项目的根目录中，鼠标右键，创建 components -> test 文件夹。

  2. 在新建的 components -> test 文件夹上，鼠标右键，点击【新建 Component】。

  3. 键入组件的名称之后回车，会自动生成组件对应的 4 个文件，后缀名分别为 .js ， .json ， .wxml 和 .wxss。

  4. 注意：为了保证目录结构的清晰，建议把不同的组件，存放到单独目录中。

使用自定义组件

🤔 组件的使用方式分为局部引用和全局引用。

  1. 在 页面.json 配置文件中引用组件的方式，叫做局部引用。

pages/contact/contact.json

pages/contact/contact.wxml

  1. 在 app.json 全局配置文件中引用组件的方式，叫做全局引用。
  1. 全局引用和局部引用的使用场景。

根据组件的使用频率和范围，来选择合适的使用方式。如果某组件在多个页面中经常被用到，建议进行全局引用；如果某组件只在特定的页面中被用到，建议进行局部引用。

组件和页面区别

  1. 组件的 .json 文件中需要声明 "component": true 属性。

  2. 组件的 .js 文件中调用的是 Component() 函数，页面的渲染调用的是 Page()。

  3. 组件的事件处理函数需要定义到 methods 节点中。

组件的数据方法和属性

定义组件数据

在小程序组件中，用于组件模板渲染的数据，需要定义到 data 属性中。

定义组件的事件处理函数

在小程序组件中，事件处理函数和自定义方法都需要定义到 methods 属性中。

定义组件 properties 属性

在小程序组件中，properties 是组件的对外属性，用来接收外界（父组件）传递到组件中的数据。需求：点击 addCount 按钮，当 count 值增加到 max 属性传入的值，就不再自增。

data 和 properties 的区别

在小程序的组件中， properties 属性和 data 数据的用法相同，它们都是可读可写的，但有如下区别。

  1. data 更倾向于存储组件的私有数据。

  2. properties 更倾向于存储外界传递到组件中的数据。

  3. 但，本质上小程序中 data 和 properties 是同一个对象的不同引用，说白了两者是完全等价的。

  4. 不建议修改 properties 数据，如果要修改，最好通过子组件通过触发父亲的自定义事件实现（后面学习）。

如何修改 properties 的值

由于 data 数据和 properties 属性在本质上没有任何区别，因此 properties 属性的值也可以用于页面渲染，也可以直接使用 setData 为 properties 中的属性重新赋值。

关于组件中的生命周期

组件的生命周期，指的是组件自身的一些函数，这些函数在特殊的时间点，或遇到一些特殊的框架事件时被自动触发。

全部生命周期函数

生命周期 参数 描述
created 无 组件实例刚刚被创建时执行
attached 无 组件实例进入页面节点树时执行
ready 无 组件在视图层布局完成后执行
moved 无 组件实例被移动到节点树另一个位置时执行
detached 无 组件实例被从页面节点树移除时执行
error Object Error 每当组件方法抛出错误时执行

主要生命周期函数

🤔 最重要的生命周期是 created、attached、detached，包含一个组件实例生命流程的最主要时间点。

  • 组件实例刚刚被创建好时，created 生命周期被触发。

    • 注意此时还不能调用 setData。

    • 通常情况下，这个生命周期只应该用于给组件 this（实例）添加一些自定义属性字段。

  • 在组件完全初始化完毕、进入页面节点树后，attached 生命周期被触发。

    • this.data 已被初始化完毕。

    • 这个生命周期很有用，绝大多数初始化工作可以在这个时机进行（例如发送请求获取数据）。

  • 在组件离开页面节点树后，detached 生命周期被触发。

    • 退出一个页面时，会触发页面内每一个自定义组件的 detached 生命周期。

    • 此时适合做一些清理性质的工作。

  • move 生命周期触发方法。

组件所在页面生命周期

是什么

有时，自定义组件的行为依赖于页面状态的变化，此时就需要用到组件所在页面的生命周期，它需要定义在 pageLifetimes 节点中，有如下 3 个。

生命周期 参数 描述
show 无 组件所在的页面被展示时执行
hide 无 组件所在的页面被隐藏时执行
resize Object Size 组件所在的页面尺寸变化时执行

🤔 页面开启 "pageOrientation": "auto" 属性，手机取消竖屏锁定，真机调试，把手机横竖切换，就会调用 resize 组件所在页面生命周期钩子函数。

掌握小程序中使用 NPM

支持和限制

目前，小程序中已经支持使用 npm 安装第三方包，从而来提高小程序的开发效率，但是在小程序中使用 npm 包有如下 3 个限制。

  1. 不支持依赖于 Node.js 内置模块的包，例如 fs、path 等。

  2. 不支持依赖于浏览器内置对象的包，例如 DOM 和 BOM 相关的 API。

  3. 不支持依赖于 C++ 插件的包。

  4. 总结：虽然 npm 上的包有千千万，但是能供小程序使用的包却“为数不多”。

Vant WeApp

  1. Vant 是有赞前端团队开源的移动端组件库，于 2016 年开源，已持续维护 6 年时间。Vant 对内承载了有赞所有核心业务，对外服务十多万开发者，是业界主流的移动端组件库之一。

  2. 扫描下方小程序二维码，体验组件库示例。

安装 Vant

在小程序项目中，安装 Vant 组件库主要分为如下几步。

  1. 通过 npm 安装，注意项目目录不能存在中文，否则会报错。

  1. 构建 npm 包。

    • 先点击【微信开发者工具】 –> 【详情】 –> 【本地设置】 –> 【使用 npm 模块】。

    • 然后点击【微信开发者工具】 –> 【菜单栏】 –> 【工具】 –> 【构建 npm】。

    • 提示“构建成功，耗时 xxx 秒”，且控制台没有任何的错误，说明包构建成功，可以进行使用。

    • 否则就需要把 node_modules、miniprogram_npm 删除，删除以后，重新安装包，并点击 【工具】 –> 【构建 npm】，进行重新构建。

    • 如果还是提示构建失败，可以将微信开发者工具重启，重启以后，运行小程序项目，查看是否还报错。

  2. 修改 app.json，将 "style": "v2", 进行移除，这可以防止小程序本身的 UI 样式和 Vant 的组件样式库冲突。

  3. 详细操作文档见 快速上手。

关于小程序 API 的 Promise 化

  1. 基于回调函数的异步 API 的缺点：小程序官方提供的异步 API 都是基于回调函数实现的，容易造成回调地狱的问题，代码的可读性、维护性差，例如网络请求的 API 需要按照如下的方式调用。
  1. API Promise 化：将官方提供的、基于回调函数的异步 API ，升级改造为基于 Promise 的异步 API ，从而提高代码的可读性、维护性，避免回调地狱的问题。

使用 miniprogram-api-promise

  1. 在小程序中，实现 API Promise 化主要依赖于 miniprogram-api-promise 这个第三方的 npm 包。

  1. 下载完成，我们不能直接使用这个包，而是需要再次重新构建 npm 包。

    • 建议在构建前先删除原有的 miniprogram_npm。

    • 然后再点击工具，构建 npm。

  2. 如果删除了 miniprogram_npm 目录，构建还是失败。

    • 需要把 node_modules、miniprogram_npm 删除。

    • 删除以后，重新安装包，重新安装以后，再次进行构建。
