分析登录的业务流程

传统登录流程：校验通过之后，直接调用接口获取 token，然后跳转到主页，而 vue-element-admin 登录流程如下。

  1. 登录表单校验通过。

  2. 调用 Vuex 提供的登录 action。

  3. 登录的 action 中会调用登录接口。

  4. 登录接口如果成功执行，会返回 token。

  5. 利用 Vuex 的特性，将 token 共享的到 Vuex 中，别的地方可以很方便访问。

  6. 登录接口会调用单独封装的请求函数。

  7. 请求函数中又会使用用到 Axios 封装的请求工具。

  8. 请求工具需要考虑区分开发环境和生产环境的问题。

  9. 请求时还要考虑前后端分离项目产生的跨域问题，可以使用代理解决。
